# Internationalization (i18n)

MySaldo is designed to be globally accessible and supports 4 languages: English, German, French, and Spanish.

## Strategy

- **Module**: `@nuxtjs/i18n`
- **Strategy**: `no_prefix` (language state managed via cookies and user profile).
- **Storage**: Locales are stored in `i18n/locales/*.json`.

## Supported Locales

- `en.json`: English (Source)
- `de.json`: German (Deutsch)
- `es.json`: Spanish (Español)
- `fr.json`: French (Français)

## Implementation Details

### Components & Pages

Use the `t()` function from `useI18n()` to translate strings.

```vue
<script setup>
const { t } = useI18n();
</script>
<template>
  <h1>{{ t('common.welcome') }}</h1>
</template>
```

### Dynamic Categories

For default categories, we store a **key** (e.g., `shopping`) in the database.
The translations for these keys are located under the `categories` namespace in the JSON files:

- `t('categories.shopping')`

Custom user-created categories store their name directly as a string, and no translation is attempted.

### Formatting Numbers & Currency

We use a custom `useCurrency.ts` composable and Nuxt's built-in locale detection to ensure currencies are formatted correctly for each language (e.g., `1.000,00 €` vs `$1,000.00`).
